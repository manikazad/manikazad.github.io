Chapter 00

The Big Picture

Why teach machines to see — and why it took us 60 years to get it right.

Why Does This Field Exist?

You open your eyes and instantly recognize thousands of objects, their textures, distances, motion, and meaning. You do this effortlessly, consuming roughly 30% of your cortex to process vision. Now imagine you want a machine to do the same thing. That's computer vision — and it's one of the hardest problems in all of AI.

Computer vision exists because the world is visual. Over 80% of the data on the internet is images and video. Self-driving cars need to see roads, medical imaging needs to detect tumors, robots need to navigate warehouses, and your phone needs to unlock when it sees your face. Without teaching machines to see, most of the AI revolution simply cannot happen.

🎯 Why Should You Care?

If you work in any domain that touches images, video, medical scans, satellite imagery, autonomous vehicles, robotics, or even natural language processing (which now uses vision-inspired architectures), CS231N gives you the foundational toolkit. The convolutional neural network architectures born from this field are the backbone of modern AI.

A 60-Year Journey

1959 — Hubel & Wiesel

Neurophysiologists discover that cat visual cortex has neurons responding to specific edge orientations. This biological insight will later inspire the architecture of convolutional networks. They found that simple cells detect edges, and complex cells pool over spatial regions — a hierarchy of increasing abstraction.

1966 — The Summer Vision Project

MIT's Seymour Papert proposed solving computer vision in one summer as a student project. The problem statement was simple: "Connect a camera to a computer and have it describe what it sees." That summer project is still unsolved 60 years later — a testament to how deceptively hard vision is.

1970s–1990s — Hand-Crafted Features

Researchers tried edge detectors (Canny, Sobel), template matching, and geometric models. David Marr proposed a computational theory of vision: from raw pixels → primal sketch (edges) → 2.5D sketch (surfaces) → 3D model. Progress was real but painfully slow.

1998 — LeNet (Yann LeCun)

LeCun builds LeNet-5, a convolutional neural network that reads handwritten digits for the US Postal Service. It works beautifully — but only on small, clean, grayscale images. The world isn't ready for bigger networks (not enough data, not enough compute).

1999–2005 — Feature Engineering Era

SIFT (1999), HOG (2005), and Bag of Words become dominant. Engineers hand-designed feature extractors, then fed those features into simple classifiers like SVMs. This was the state-of-the-art for a decade. You had to be a domain expert to design good features.

2009 — ImageNet

Fei-Fei Li and her team create ImageNet: 14 million labeled images across 22,000 categories. For the first time, there's enough data to train very large models. The annual ImageNet Large Scale Visual Recognition Challenge (ILSVRC) begins.

2012 — The AlexNet Moment

Alex Krizhevsky, Ilya Sutskever, and Geoffrey Hinton enter ILSVRC with AlexNet — a deep convolutional neural network trained on GPUs. They crush the competition, dropping the error rate from 26% to 16%. This is the moment everything changed. Deep learning had arrived.

2015+ — Superhuman Performance

ResNet achieves 3.57% error on ImageNet — better than humans (~5%). Vision Transformers (ViTs) arrive. Foundation models like CLIP learn vision and language together. The field explodes into detection, segmentation, generation, and 3D understanding.

What If Neural Networks Hadn't Worked?

If deep learning hadn't emerged, you'd still be in the feature engineering era. You'd manually design SIFT descriptors, HOG templates, and color histograms. You'd need years of domain expertise to build a passable image classifier. Each new task (faces, cars, medical images) would require starting from scratch with new hand-designed features.

The breakthrough of deep learning is that the network learns its own features. Instead of you telling the computer "look for edges, then corners, then shapes," the network discovers these representations automatically from data. The first layers learn edges, middle layers learn textures and parts, and deep layers learn whole objects. This is representation learning, and it's the central idea of CS231N.

💡 Core Insight

CS231N is fundamentally about one idea: learning hierarchical representations directly from pixels. Every chapter builds toward understanding how neural networks transform raw pixel values into meaningful, abstract features that enable classification, detection, generation, and understanding.

What You'll Build, Concept by Concept

This course walks you through building a complete visual intelligence system from scratch. We start with the simplest possible classifier — comparing pixel values directly — and show why that fails. Then we introduce linear classifiers, loss functions, and optimization. We discover that linear models can't capture complex patterns, so we stack them into neural networks. We learn backpropagation to train them efficiently. Then we discover convolutions — the key architectural insight from neuroscience — and build modern deep networks. Finally, we explore the frontiers: object detection, image generation, sequence modeling, and attention.

At every step, we'll ask: Why this approach? What problem does it solve? What came before? What would you use instead?

Chapter 01

Image Classification

The core task of visual recognition — assigning a label to an image — and why it's absurdly hard for machines.

Why Does Image Classification Exist?

At its heart, image classification answers one question: "What is in this image?" It's the foundation upon which all other vision tasks are built. Object detection is "what AND where." Segmentation is "which pixel belongs to what." Image generation is "create new images of this category." But classification came first, and getting it right was the gateway to everything else.

🎯 The Fundamental Problem

To a computer, an image is just a 3D array of numbers. A 256×256 color image is a matrix of shape (256, 256, 3) containing integers from 0 to 255. The challenge: given this grid of numbers, determine whether it contains a cat, a dog, a car, or any of thousands of possible categories. The gap between "grid of numbers" and "semantic meaning" is called the semantic gap, and bridging it is the central challenge of computer vision.

What Makes This So Hard?

Consider what happens when you shift a cat one pixel to the right. Every single number in the array changes. Yet it's obviously still a cat. This is just one of many challenges:

Viewpoint Variation

The same object looks completely different from the front, side, top, or at an angle. A car seen head-on vs. from above shares almost no pixel patterns.

Illumination

The same scene under sunlight vs. shadow vs. artificial light produces wildly different pixel values. Your classifier must be invariant to lighting.

Deformation

Cats can curl, stretch, twist into bizarre shapes. A classifier must recognize a cat whether it's sleeping, jumping, or squeezing into a box.

Occlusion

Objects are often partially hidden. You need to recognize a cat even if only its head peeks out from behind a sofa.

The Naive Approach: Why Can't We Just Write Rules?

Your first instinct might be to write explicit rules: "If there are pointy ears at the top AND whiskers AND fur texture, it's a cat." This approach — called explicit programming — fails catastrophically. Why?

Because you can't write rules for every viewpoint, every lighting condition, every breed, every pose. And even if you could write rules for cats, you'd need entirely different rules for dogs, cars, birds, and 10,000 other categories. The rule-based approach doesn't scale.

📜 Historical Context

The early vision community (1960s–1990s) tried exactly this approach. They built edge detectors, shape matchers, and geometric models. Some worked for narrow domains (reading zip codes, detecting faces under controlled lighting) but always failed when conditions changed. The realization was that you need a data-driven approach: instead of writing rules, show the computer thousands of examples and let it figure out the patterns.

The Data-Driven Paradigm

The modern approach to image classification follows a simple recipe:

1. Collect a dataset of labeled images (e.g., ImageNet: 1.2 million images, 1,000 categories).

2. Train a classifier on this dataset — the model examines many examples and learns patterns.

3. Evaluate on new, unseen images to see if the learned patterns generalize.

This is the machine learning approach, and it's the foundation of everything in CS231N.

Attempt #1: Nearest Neighbor Classifier

The simplest possible data-driven classifier: given a new image, find the most similar image in the training set and copy its label. "Similarity" is measured by the distance between pixel arrays.

L1 (Manhattan) Distance

d₁(I₁, I₂) = Σₚ |I₁ᵖ - I₂ᵖ| Sum of absolute differences across all pixels.

L2 (Euclidean) Distance

d₂(I₁, I₂) = √(Σₚ (I₁ᵖ - I₂ᵖ)²) Root of the sum of squared differences.

Why Nearest Neighbor Fails

The nearest neighbor approach has fatal flaws for real images:

Test time is O(n) — you must compare against every training image. With 1 million training images, that's impossibly slow.

Pixel distance is meaningless — An image shifted by one pixel has a huge pixel distance but is semantically identical. Meanwhile, two very different images can have small pixel distances (imagine a white cat and a white dog).

Curse of dimensionality — To densely cover the space of all possible images, you'd need exponentially more data as dimensions grow. A 256×256×3 image lives in a ~200,000-dimensional space. You'd need more training images than atoms in the universe to cover it.

💡 The Lesson

Nearest neighbor teaches us the critical insight: we need to compare images in feature space, not pixel space. The job of a neural network is to learn a transformation from pixel space to a feature space where semantically similar images are close together and different images are far apart. This is the essence of representation learning.

Quick Check: Why does comparing raw pixel values fail for image classification?
Pixel-level similarity is dominated by superficial features like brightness, color, and translation. Two images of the same object under different lighting can have enormous pixel distances, while images of different objects can be pixel-similar. We need to compare in a learned feature space where semantic meaning is captured.
Chapter 02

Linear Classifiers

The simplest parametric model — and the building block of every neural network.

Why Linear Classifiers?

Nearest neighbor stored all training data and compared at test time. That's a non-parametric approach — the "model" IS the data. A linear classifier takes the opposite approach: it compresses everything it learned into a small set of parameters (weights and biases). At test time, classification is a single matrix multiplication — blazingly fast.

🎯 If Linear Classifiers Didn't Exist

Without parametric models, you'd be stuck with methods that scale with your training set size. Nearest neighbor needs all data at test time. Decision trees grow with data complexity. Linear classifiers introduced the idea of learning a compact set of weights that summarize patterns — this principle extends to neural networks with millions of parameters that still make predictions in milliseconds.

The Model: f(x, W) = Wx + b

A linear classifier computes a score for each class as a linear function of the input pixels:

f(x, W) = Wx + b x = input image, flattened to a vector (e.g., 32×32×3 = 3072 dimensions) W = weight matrix (num_classes × dimensions, e.g., 10 × 3072) b = bias vector (num_classes, e.g., 10) Output: one score per class

Each row of W is a template for one class. The dot product Wx measures how well the image matches each template. The class with the highest score wins.

The Geometric View

Each row of W defines a hyperplane in pixel space. The classifier draws linear decision boundaries between classes. Points on one side of a hyperplane are classified as one class; points on the other side are another class.

Interactive: Linear Decision Boundaries in 2D
Drag the slider to rotate the decision boundary. In "XOR Problem" mode, notice how NO linear boundary can separate the classes — this is a fundamental limitation that motivates neural networks.

The Template View

Each row of W can be reshaped back into an image. This "template" is what the classifier thinks each class looks like on average. For CIFAR-10, the learned templates show blurry, averaged versions of each category — a horse template shows a brownish blob facing both left and right because horses can face either direction. This averaging is a fundamental limitation: one template per class can't capture the diversity of real objects.

Why Linear Classifiers Matter

Despite their simplicity, linear classifiers are essential because:

They're the building block of neural networks. Every layer of a neural network is a linear transformation followed by a nonlinearity. Understanding linear classifiers deeply is understanding the atom of deep learning.

They're fast. Both training and inference are simple matrix operations. They set the baseline that more complex models must beat.

They reveal limitations clearly. When a linear classifier fails (XOR, multi-modal distributions), it points exactly to why we need nonlinear models.

💡 Key Insight

A neural network is just stacked linear classifiers with nonlinearities between them. The nonlinearities allow the network to learn non-linear decision boundaries. Remove the nonlinearities, and a 100-layer network collapses into a single linear classifier — the composition of linear functions is still linear.

Chapter 03

Loss Functions

How do we tell a model it's wrong — and by exactly how much?

Why Do Loss Functions Exist?

A model makes predictions. But how do we know if those predictions are good? We need a function that takes the model's output and the correct answer, and returns a single number indicating how badly the model is doing. This is the loss function (also called cost function or objective function).

Without a loss function, there's no way to train. The entire learning process is: make a prediction → compute the loss → adjust weights to reduce the loss → repeat. The loss function is the compass that guides learning.

📜 Origins

Loss functions come from statistics and optimization theory. The concept of "least squares" (minimizing squared errors) dates back to Gauss and Legendre in the early 1800s. Maximum likelihood estimation, which leads to cross-entropy loss, was formalized by R.A. Fisher in 1922. These mathematical tools are centuries old — deep learning repurposed them brilliantly.

SVM Loss (Hinge Loss)

The Support Vector Machine loss says: "The correct class score should be higher than every incorrect class score by at least a margin Δ (usually 1)."

Lᵢ = Σⱼ≠yᵢ max(0, sⱼ - syᵢ + Δ) sⱼ = score for class j syᵢ = score for the correct class Δ = margin (typically 1) If correct class score beats all others by the margin → loss is 0 Otherwise → loss is proportional to the violation

Think of it as: "I'm happy when the correct class wins by a comfortable margin. I'm unhappy proportionally to how much any wrong class comes close or exceeds the correct class."

Softmax / Cross-Entropy Loss

While SVM loss only cares about margins between scores, the Softmax classifier interprets scores as (unnormalized) log-probabilities and wants to maximize the probability of the correct class.

P(Y = k | X = xᵢ) = eˢᵏ / Σⱼ eˢⱼ ← softmax function Lᵢ = -log P(Y = yᵢ | X = xᵢ) ← cross-entropy loss = -log(eˢʸⁱ / Σⱼ eˢⱼ) This loss is 0 when P(correct class) = 1 (perfect confidence) and increases toward infinity as confidence in the correct class → 0

SVM Loss

Pros: Simple, interpretable margin. Once margin is satisfied, loss is exactly 0 — no further pressure.

Cons: Scores aren't probabilities. Doesn't push for high-confidence predictions. Less smooth gradient landscape.

Use when: You want binary-like decisions with a clear margin. Classic SVMs are mathematically elegant.

Softmax / Cross-Entropy

Pros: Outputs are probabilities (sum to 1). Always pushes for improvement. Smooth, well-behaved gradients.

Cons: Never fully satisfied — always wants higher confidence. Can lead to overconfident predictions.

Use when: You want probability outputs. This is the default for most modern deep networks.

Regularization: The Weight Penalty

The total loss isn't just the data loss — we add a regularization term that penalizes complex models:

L = (1/N) Σᵢ Lᵢ(f(xᵢ, W), yᵢ) + λR(W) Data loss (fit the data) + Regularization (keep model simple) λ = regularization strength (hyperparameter)

Common regularization: L2 regularization R(W) = Σ W² spreads weights out evenly. L1 regularization R(W) = Σ |W| promotes sparsity (many weights become exactly zero). We'll explore this in depth in Chapter 11.

Chapter 04

Optimization

How to find the best weights — navigating a landscape you can't see, one step at a time.

Why Optimization Exists

You have a loss function that measures how bad your model is. You have weights that control the model's behavior. Optimization is the process of finding weights that minimize the loss. Without optimization, you have a model that can't learn — it's just a random function.

Think of it as hiking in fog: you're on a mountain (the loss landscape) and want to reach the lowest valley. You can't see the landscape, but you can feel the slope under your feet. Optimization is the strategy for taking steps downhill.

Strategy 1: Random Search (Terrible)

Generate random weight matrices, compute the loss for each, keep the best one. This sounds absurd — and it is — but it establishes a baseline. On CIFAR-10 with 10 classes, random chance gives ~10% accuracy. Random search achieves ~15.5%. We can do much better.

Strategy 2: Follow the Gradient

The gradient of the loss with respect to the weights tells you the direction of steepest increase. Go in the opposite direction to decrease the loss. This is gradient descent.

W_new = W_old - learning_rate × ∇L(W) ∇L(W) = gradient of loss with respect to W learning_rate (α) = step size (how far to go in each step) Too small α → painfully slow convergence Too large α → overshoot and diverge Just right α → smooth convergence to a minimum

Stochastic Gradient Descent (SGD)

Computing the gradient over the entire dataset (millions of images) is expensive. Instead, SGD approximates the gradient using a small random minibatch (32, 64, 128 images). The gradient estimate is noisy but unbiased, and the noise actually helps escape local minima.

Beyond Vanilla SGD

SGD + Momentum

Add a "velocity" term that accumulates past gradients. Like a ball rolling downhill, momentum carries you through flat regions and dampens oscillations in ravines.

v = μ × v - lr × ∇L(W) ← update velocity (μ ~ 0.9) W = W + v ← update weights using velocity

Adam (Adaptive Moment Estimation)

The most popular optimizer in deep learning. It combines momentum with per-parameter adaptive learning rates. Parameters that have had large gradients get smaller steps; parameters with small gradients get larger steps.

m = β₁ × m + (1-β₁) × ∇L ← first moment (mean of gradients) v = β₂ × v + (1-β₂) × (∇L)² ← second moment (mean of squared gradients) W = W - lr × m̂ / (√v̂ + ε) ← adaptive update β₁ = 0.9, β₂ = 0.999, ε = 1e-8 (defaults that work almost everywhere)
💡 Which Optimizer Should You Use?

Start with Adam (lr=3e-4 for most tasks). It's robust and works well out of the box. If you need the absolute best performance and can afford hyperparameter tuning, SGD + Momentum often achieves slightly better final accuracy but requires careful learning rate scheduling. In practice, Adam gets you 95% of the way with 10% of the tuning effort.

Chapter 05

Backpropagation

The engine of deep learning — how gradients flow backward through a computation graph.

Why Backpropagation Exists

Gradient descent needs gradients. For a linear classifier with 3,072 parameters, you could compute each gradient numerically (perturb each weight, measure change in loss). But a modern neural network has millions of parameters. Computing each gradient individually would be impossibly slow.

Backpropagation solves this by computing all gradients simultaneously using the chain rule of calculus. It's not a learning algorithm — it's a method for efficiently computing gradients. Combined with gradient descent, it enables training of deep networks.

📜 A Tumultuous History

The chain rule has been known since Leibniz (1600s). Backpropagation was independently discovered multiple times: by Linnainmaa (1970) for automatic differentiation, by Werbos (1974) in his PhD thesis, and popularized by Rumelhart, Hinton & Williams (1986) who showed it could train multi-layer networks. For decades, people thought it wouldn't work for deep networks. It took until 2006–2012, with better initializations, architectures, and GPUs, before deep backprop-trained networks finally succeeded.

The Computation Graph

Every mathematical computation can be broken into a graph of simple operations. Backpropagation works by:

Forward pass: Compute the output, saving intermediate values at each node.

Backward pass: Starting from the loss, propagate gradients backward through the graph using the chain rule.

Chain rule: ∂L/∂x = (∂L/∂z) × (∂z/∂x) "The gradient of the loss with respect to x equals the gradient of the loss with respect to z (the output) times the local gradient of z with respect to x." Each node only needs to know: 1. Its local gradient (∂output/∂input) 2. The upstream gradient (∂L/∂output) Then it computes: downstream gradient = upstream × local

Patterns in Gradient Flow

As you work through many computation graphs, patterns emerge:

Add gate: Distributes the gradient equally to both inputs. Both inputs get the same upstream gradient.

Multiply gate: Swaps and scales. If z = x × y, then ∂z/∂x = y and ∂z/∂y = x. Each input's gradient is the upstream gradient scaled by the other input's value.

Max gate: Routes the gradient. Only the input that "won" the max gets the gradient; the other gets zero.

Sigmoid gate: Has a beautifully simple local gradient: σ(x)(1 - σ(x)). This becomes very small for large |x|, causing the "vanishing gradient" problem.

💡 The Big Picture

Backpropagation is just organized application of the chain rule. Modern deep learning frameworks (PyTorch, TensorFlow) build computation graphs automatically and handle backpropagation for you. But understanding it deeply is essential because: (1) you'll debug gradient issues (vanishing/exploding), (2) you'll understand why certain architectures work, and (3) you'll know what's happening under the hood when you call loss.backward().

Chapter 06

Neural Networks

Stacking linear classifiers with nonlinearities to learn any function.

Why Do Neural Networks Exist?

Linear classifiers can only draw straight lines (hyperplanes) between classes. Real-world data is almost never linearly separable. You need curves, loops, and complex boundaries. Neural networks exist because stacking linear transformations with nonlinear activation functions creates a model that can approximate any continuous function — the Universal Approximation Theorem.

🎯 What Would You Use Instead?

Before neural networks dominated, people used: kernel SVMs (transform data to high-dimensional space where it's linearly separable), Random Forests (ensemble of decision trees), and Boosting (sequentially correcting weak learners). These methods still work well for small datasets and tabular data. But for images, audio, and text at scale, neural networks are overwhelmingly superior because they learn features automatically while other methods require hand-crafted features.

From Linear to Multi-Layer

A 2-layer neural network:

Hidden layer: h = f(W₁x + b₁) ← linear transform + activation Output layer: s = W₂h + b₂ ← linear transform to class scores f = nonlinear activation function (ReLU, sigmoid, tanh) W₁ ∈ ℝ^(H×D), W₂ ∈ ℝ^(C×H) H = number of hidden neurons (width of the network)

The magic is in the activation function f. Without it, W₂(W₁x) = (W₂W₁)x = W'x — just another linear classifier. The nonlinearity between layers lets each layer learn progressively more abstract features.

The Universal Approximation Theorem

A neural network with a single hidden layer containing enough neurons can approximate any continuous function to arbitrary precision. This was proven independently by Cybenko (1989) and Hornik (1991). But there's a catch: "enough neurons" might mean an astronomically large number. In practice, deeper networks are exponentially more efficient than wide shallow ones — they can represent the same functions with far fewer parameters. This is why we use deep learning, not wide learning.

Chapter 07

Activation Functions

The nonlinear spark that gives neural networks their power.

Why Activation Functions?

Without activation functions, a neural network is just matrix multiplication — a linear model no matter how many layers. Activation functions introduce nonlinearity, enabling the network to learn curved decision boundaries and complex representations.

Interactive: Activation Function Explorer
Each activation function and its derivative (gradient). The derivative determines how well gradients flow during backpropagation. Notice sigmoid's gradient vanishes for large |x| — the "vanishing gradient" problem.
Sigmoid: σ(x) = 1/(1+e⁻ˣ)

Squashes input to (0, 1). Historically important but largely abandoned because: (1) gradients vanish for large |x|, (2) outputs aren't zero-centered, causing zigzag gradient updates, (3) exp() is expensive.

ReLU: f(x) = max(0, x)

The modern default. Gradient is 1 for x > 0, 0 for x < 0. 6× faster convergence than sigmoid. But "dead ReLU" problem: if a neuron's input is always negative, it permanently dies (gradient = 0 forever).

Leaky ReLU: f(x) = max(0.01x, x)

Fixes dead ReLU by allowing a small gradient (0.01) for negative values. The neuron never fully dies. Parametric ReLU (PReLU) learns the slope.

GELU / Swish

Modern smooth alternatives. GELU ≈ x × σ(1.702x). Used in Transformers and modern architectures. Slightly better performance in practice but more expensive to compute.

💡 Practical Advice

Use ReLU as your default. It's simple, fast, and works well. If you see dying neurons, switch to Leaky ReLU. For Transformers and modern architectures, GELU is standard. Don't use sigmoid or tanh in hidden layers — they cause vanishing gradients. Sigmoid is fine for the output layer when you need probabilities between 0 and 1.

Chapter 08

Convolutional Neural Networks

The architectural breakthrough inspired by neuroscience that made visual recognition work.

Why Convolutions Exist

A fully-connected layer on a 224×224×3 image needs 224×224×3 = 150,528 weights per neuron. With 1,000 neurons in the first hidden layer, that's 150 million parameters — just for one layer. This is absurd: most of those connections are useless (why should a neuron looking at the top-left corner care about pixel values in the bottom-right?), and with so many parameters, the model will catastrophically overfit.

Convolutions solve this by exploiting three key insights from biology and common sense:

Local Connectivity

A neuron only looks at a small local region of the input (its receptive field), not the entire image. Edges, textures, and patterns are local phenomena.

Weight Sharing

The same set of weights (the filter/kernel) is applied at every spatial position. An edge detector useful at position (10,10) is equally useful at position (200,200).

Translation Equivariance

If the input shifts, the output shifts by the same amount. The network detects features regardless of where they appear in the image.

Hierarchical Features

Stack conv layers: first detects edges → second detects textures from edges → third detects parts from textures → deeper layers detect whole objects.

How Convolution Works

A convolutional filter is a small weight matrix (e.g., 3×3) that slides across the input image. At each position, it computes a dot product between the filter and the local image patch, producing one output value. Slide across the entire image to produce a 2D activation map. Different filters detect different patterns.

Input: H × W × C (e.g., 32 × 32 × 3) Filter: F × F × C (e.g., 3 × 3 × 3 = 27 weights + 1 bias) Output: H' × W' × 1 (one activation map per filter) With K filters → output is H' × W' × K H' = (H - F + 2P) / S + 1 W' = (W - F + 2P) / S + 1 P = padding (typically P = F/2 to preserve spatial size) S = stride (step size; stride 2 → halve spatial dimensions)

Pooling Layers

Max pooling reduces spatial dimensions by taking the maximum value in each local region (typically 2×2 with stride 2, halving the resolution). This provides a degree of translation invariance and reduces computation. Average pooling takes the mean instead. Modern architectures sometimes replace pooling with strided convolutions.

A Typical CNN Architecture

INPUT → [CONV → RELU → CONV → RELU → POOL] × N → FC → FC → SOFTMAX Spatial resolution shrinks: 224 → 112 → 56 → 28 → 14 → 7 Channel depth grows: 3 → 64 → 128 → 256 → 512 Total parameters: ~millions (vs. billions for fully-connected)
💡 Why CNNs Changed Everything

By baking in the assumptions of local connectivity and weight sharing, CNNs reduced parameter count by orders of magnitude compared to fully-connected networks. This made it possible to train deep networks on images without catastrophic overfitting. The hierarchy of learned features — edges → textures → parts → objects — mirrors how the biological visual cortex processes information. This isn't a coincidence: Hubel and Wiesel's discovery of simple and complex cells directly inspired this architecture.

Chapter 09

CNN Architectures

The evolution from AlexNet to ResNet — how architecture design drove the deep learning revolution.

The Architecture Timeline

1998 — LeNet-5 (Yann LeCun)

5 layers, 60K parameters. Solved digit recognition (MNIST). Architecture: Conv → Pool → Conv → Pool → FC → FC → Output. Established the fundamental CNN pattern but was limited to small grayscale images.

2012 — AlexNet (Krizhevsky et al.)

8 layers, 60M parameters. Won ILSVRC 2012 by a landslide. Key innovations: ReLU activation (instead of sigmoid), dropout regularization, GPU training (two GTX 580s), data augmentation. This paper is the "Big Bang" of modern deep learning.

2014 — VGGNet (Simonyan & Zisserman)

16–19 layers, 138M parameters. Key insight: replace large filters (11×11, 5×5) with stacks of 3×3 filters. Three 3×3 layers have the same receptive field as one 7×7 layer but with more nonlinearities and fewer parameters. Simplicity wins.

2014 — GoogLeNet / Inception (Szegedy et al.)

22 layers, only 5M parameters. Introduced the Inception module: process the input through multiple filter sizes (1×1, 3×3, 5×5) in parallel and concatenate results. 1×1 convolutions reduce channel depth before expensive operations (the "bottleneck" trick).

2015 — ResNet (He et al.)

Up to 152 layers, yet easy to train. The breakthrough: skip connections (residual connections). Instead of learning f(x), learn f(x) = H(x) - x (the residual), so the output is x + f(x). This lets gradients flow directly through skip connections, solving the vanishing gradient problem in very deep networks.

2020+ — Vision Transformers (ViT)

Apply the Transformer architecture (from NLP) directly to image patches. Split image into 16×16 patches, embed them as tokens, process with self-attention. With enough data, ViTs match or exceed CNN performance. Architectures like Swin Transformer add local window attention for efficiency.

Interactive: ResNet Skip Connection
Compare gradient flow in plain vs. residual networks. Without skip connections, gradients vanish exponentially with depth. With skip connections, gradients have a direct highway back to early layers.
💡 Which Architecture Should You Use?

For most practical purposes: ResNet-50 is the workhorse (great accuracy, reasonable compute). For mobile/edge: MobileNet or EfficientNet. For state-of-the-art: ConvNeXt (modernized CNN) or Swin Transformer. Always start with a pretrained model and fine-tune — training from scratch is rarely justified unless you have millions of images.

Chapter 10

Training Deep Networks

The dark arts of making deep networks actually learn — initialization, normalization, and hyperparameters.

Why Training is Hard

A neural network with random weights will produce random outputs. Training should gradually improve these outputs — but deep networks are notoriously finicky. Without careful setup, training can fail completely: losses explode to infinity, gradients vanish to zero, or the network learns nothing useful. This chapter covers the techniques that make training reliable.

Weight Initialization

If you initialize all weights to zero, every neuron computes the same thing — they all learn identical features forever (the symmetry problem). If you initialize with values too large, activations saturate and gradients vanish. Too small, and activations shrink to zero through layers.

Xavier/Glorot Initialization (2010)

W ~ N(0, σ²) where σ = √(2 / (nᵢₙ + nₒᵤₜ)) Designed for sigmoid/tanh: maintains variance of activations across layers. If variance grows or shrinks, signals get amplified or attenuated, causing training to fail.

He Initialization (2015)

W ~ N(0, σ²) where σ = √(2 / nᵢₙ) Designed for ReLU: accounts for the fact that ReLU kills half the activations (negative values → 0), so we need 2× the variance compared to Xavier.

Batch Normalization

Perhaps the most important training innovation after backpropagation itself. BatchNorm normalizes the activations of each layer to have zero mean and unit variance within each minibatch.

Given a minibatch B = {x₁, ..., xₘ}: μ_B = (1/m) Σ xᵢ ← batch mean σ²_B = (1/m) Σ (xᵢ - μ_B)² ← batch variance x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε) ← normalize yᵢ = γ × x̂ᵢ + β ← scale and shift (learned) γ and β are learnable parameters that allow the network to undo the normalization if that's optimal.

Why it works: It reduces internal covariate shift — the distribution of each layer's inputs changes during training as the preceding layers change. BatchNorm stabilizes these distributions, allowing higher learning rates and faster convergence. It also acts as a mild regularizer because each example's normalization depends on the random minibatch.

Learning Rate Scheduling

Starting with a high learning rate and decaying it during training is critical for achieving best performance:

Step decay: Reduce LR by a factor (0.1) at predetermined epochs (e.g., at epoch 30, 60, 90).

Cosine annealing: Smoothly decay LR following a cosine curve from initial value to near zero.

Warmup: Start with a very small LR and linearly increase it for the first few epochs, then apply decay. Essential for large batch training.

Data Augmentation

Artificially increase your training set by applying random transformations to images: horizontal flips, random crops, color jittering, rotation, scaling. This is essentially free regularization — it teaches the model invariance to these transformations without any additional real data.

💡 Training Recipe Checklist

1. Use He initialization with ReLU. 2. Add BatchNorm after each conv/linear layer. 3. Start with Adam (lr=3e-4) or SGD+Momentum (lr=0.1) with cosine schedule. 4. Apply aggressive data augmentation. 5. Use a validation set to tune hyperparameters. 6. Train until validation loss stops improving (early stopping). 7. If overfitting, add more regularization (dropout, weight decay, more augmentation). If underfitting, increase model capacity.

Chapter 11

Regularization

Preventing overfitting — the art of helping your model generalize beyond the training data.

Why Regularization Exists

A sufficiently large neural network can memorize the entire training set — achieving 100% training accuracy — while performing terribly on new data. This is overfitting: the model has learned noise and idiosyncrasies of the training data rather than general patterns. Regularization is every technique that reduces overfitting and improves generalization.

🎯 The Bias-Variance Tradeoff

Bias: Error from underfitting (model too simple). Variance: Error from overfitting (model too complex). Regularization adds bias (makes the model slightly simpler) to dramatically reduce variance. The sweet spot gives the lowest total error on unseen data.

Weight Decay (L2 Regularization)

Add the squared magnitude of weights to the loss: L_total = L_data + λ × Σ W². This penalizes large weights, pushing the model toward simpler, smoother functions. Equivalent to decaying weights toward zero at each update step (hence "weight decay"). Use λ between 1e-4 and 1e-2.

Dropout

During training, randomly set each neuron's activation to zero with probability p (typically 0.5 for FC layers, 0.1–0.3 for conv layers). This forces redundancy — no neuron can rely on any other specific neuron being present. At test time, use all neurons but scale activations by (1-p) to compensate (or use inverted dropout: scale by 1/(1-p) during training).

0.50
Each resample randomly drops different neurons (shown as faded). This creates a different "sub-network" each time, effectively training an ensemble of networks.

Other Regularization Techniques

Early stopping: Stop training when validation loss starts increasing. Simple and effective.

Data augmentation: Artificial expansion of training data through transformations (flips, crops, color shifts, MixUp, CutOut, CutMix). Often the single most effective regularizer.

Label smoothing: Instead of hard labels (1 for correct, 0 for incorrect), use soft labels (0.9 for correct, 0.1/K for incorrect). Prevents overconfident predictions.

Stochastic depth: Randomly skip entire residual blocks during training. Training speedup + regularization.

Chapter 12

Object Detection & Segmentation

Beyond "what" — answering "where" and "which pixels" belong to each object.

Why Detection Exists

Classification tells you an image contains a dog. But where? How many dogs? What else is in the scene? Object detection draws bounding boxes around every object and labels each one. Semantic segmentation labels every pixel with its class. Instance segmentation does both — it labels each pixel AND distinguishes between different instances of the same class.

Object Detection

Output: set of (class, x, y, width, height, confidence) tuples. "There's a dog at position (100, 200) of size 150×200 with 95% confidence."

Semantic Segmentation

Output: H×W label map. Every pixel gets a class: road, car, pedestrian, sky. No distinction between individual instances.

Instance Segmentation

Output: per-pixel labels + instance IDs. "These pixels are car #1, these are car #2." Mask R-CNN is the landmark approach.

Panoptic Segmentation

Unified: semantic labels for stuff (sky, road) + instance labels for things (cars, people). Complete scene understanding.

Object Detection Approaches

Two-Stage Detectors: R-CNN Family

R-CNN (2014): Generate ~2,000 region proposals (selective search) → warp each to fixed size → classify with CNN. Slow (~47 seconds per image).

Fast R-CNN (2015): Share CNN computation across proposals by processing the whole image once and extracting features for each region. ~0.3 sec/image.

Faster R-CNN (2015): Replace selective search with a learned Region Proposal Network (RPN) that shares conv features with the detector. ~0.2 sec/image. The RPN uses anchor boxes — predefined shapes at each spatial position — and predicts which anchors contain objects and how to refine their coordinates.

One-Stage Detectors: YOLO, SSD

YOLO ("You Only Look Once"): Divide image into grid, predict boxes and classes directly per cell. Much faster (~45 FPS) but initially less accurate. Modern versions (YOLOv5+) close the accuracy gap.

SSD (Single Shot Detector): Detect at multiple scales using feature maps from different layers. Fast and accurate.

Semantic Segmentation

The key architecture is the Fully Convolutional Network (FCN): replace the final FC layers of a classification CNN with conv layers, then upsample back to full resolution. The encoder (downsampling path) extracts features; the decoder (upsampling path) recovers spatial resolution.

U-Net adds skip connections between encoder and decoder at matching resolutions, combining high-resolution spatial information with deep semantic features. Originally designed for biomedical image segmentation, U-Net is now ubiquitous.

Chapter 13

Generative Models

From recognizing images to creating them — learning the distribution of visual data.

Why Generative Models?

So far, everything has been discriminative: given an image, predict its class (P(y|x)). Generative models flip this: they learn the data distribution P(x) and can create new samples. Why care? Because (1) data augmentation: generate synthetic training data; (2) understanding: if you can generate realistic images, you deeply understand image structure; (3) creativity: art, design, content creation; (4) anomaly detection: detect when something doesn't fit the learned distribution.

Generative Adversarial Networks (GANs)

Invented by Ian Goodfellow in 2014. Two networks competing in a game:

Generator G: Takes random noise z and produces a fake image G(z). Goal: fool the discriminator.

Discriminator D: Takes an image and predicts whether it's real or fake. Goal: catch the generator.

min_G max_D E[log D(x)] + E[log(1 - D(G(z)))] The generator minimizes; the discriminator maximizes. At equilibrium: G produces perfect fakes, D can't tell real from fake (outputs 0.5 for everything).

GANs produce stunningly realistic images but are notoriously hard to train: mode collapse (generator produces only a few types of images), training instability, and no good way to measure quality.

Variational Autoencoders (VAEs)

Encode input image to a distribution in latent space (mean and variance), sample from that distribution, and decode back to an image. Trained to minimize reconstruction loss + KL divergence (keeping the latent distribution close to a standard Gaussian).

VAEs produce blurrier images than GANs but are more stable to train and give a meaningful latent space where you can interpolate between images.

Diffusion Models

The modern state-of-the-art (DALL-E 2, Stable Diffusion, Midjourney). The idea: gradually add noise to an image until it becomes pure Gaussian noise (forward process). Then train a network to reverse this process — to denoise step by step (reverse process). Generate new images by starting from pure noise and repeatedly denoising.

Diffusion models produce the highest quality images, are stable to train, and offer fine-grained control through text conditioning. They've largely superseded GANs for image generation.

💡 The Generative Model Landscape

GANs: Fast sampling, sharp images, but hard to train and no density estimation. VAEs: Stable training, meaningful latent space, but blurry outputs. Diffusion: Best quality, stable training, but slow sampling (many denoising steps). Flow models: Exact density estimation, but constrained architectures. In 2024+, diffusion models dominate, but the field moves fast.

Chapter 14

Recurrent Neural Networks

Processing sequences — from image captioning to video understanding.

Why RNNs Exist

Everything so far processes fixed-size inputs and produces fixed-size outputs. But many tasks involve sequences: image captioning (image → sequence of words), video classification (sequence of frames → label), machine translation (sequence → sequence). RNNs handle variable-length sequences by maintaining a hidden state that carries information across time steps.

hₜ = f(Wₓₕ × xₜ + Wₕₕ × hₜ₋₁ + b) hₜ = hidden state at time t xₜ = input at time t Wₓₕ = input-to-hidden weights (shared across all time steps) Wₕₕ = hidden-to-hidden weights (shared across all time steps) f = activation function (tanh)

The Vanishing Gradient Problem

RNNs struggle with long sequences. During backpropagation through time, gradients are multiplied by Wₕₕ at each step. If the largest eigenvalue of Wₕₕ is < 1, gradients shrink exponentially → the network forgets long-range dependencies. If > 1, gradients explode → training diverges.

LSTM and GRU: Gated Architectures

Long Short-Term Memory (LSTM) solves vanishing gradients with a cell state and three gates: forget gate (what to discard), input gate (what to add), and output gate (what to output). The cell state acts like a highway, allowing gradients to flow unchanged across many time steps.

Gated Recurrent Unit (GRU) is a simpler variant with two gates (reset and update) that performs comparably to LSTM with fewer parameters.

Applications in Vision

Image Captioning: CNN encodes image → LSTM generates caption word by word, conditioned on the image features.

Visual Question Answering: CNN encodes image, RNN encodes question → combined to produce an answer.

Video Understanding: Process sequences of CNN features with RNNs to classify actions or predict future frames.

💡 Are RNNs Still Relevant?

Transformers have largely replaced RNNs for most sequence tasks due to their ability to process all positions in parallel (vs. RNN's sequential processing) and better long-range dependency modeling through attention. However, understanding RNNs is still valuable: they're simpler to grasp, computationally efficient for short sequences, and the gating ideas (LSTM, GRU) influenced Transformer design. Some modern architectures (state space models like Mamba) revive RNN-like ideas with better parallelism.

Chapter 15

Attention & Transformers

The architecture that unified NLP and vision — attending to what matters.

Why Attention Exists

In an RNN-based encoder-decoder (e.g., for translation), the entire input sequence must be compressed into a single fixed-size vector. This is a bottleneck — long sequences lose information. Attention (Bahdanau et al., 2015) solves this by letting the decoder look back at all encoder states and attend to the most relevant ones at each decoding step.

Self-Attention

The Transformer (Vaswani et al., 2017, "Attention Is All You Need") uses self-attention: each position in a sequence attends to all other positions, computing a weighted sum based on relevance.

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) × V Q = queries (what am I looking for?) K = keys (what do I contain?) V = values (what information do I carry?) Each position computes: 1. Score = dot product of my query with every key 2. Weight = softmax of scores (normalized attention) 3. Output = weighted sum of values Multi-head attention: run h parallel attention operations with different learned Q, K, V projections, then concatenate.

The Transformer Architecture

Stack of identical blocks, each containing:

1. Multi-head self-attention — each position attends to all others.

2. Feed-forward network (FFN) — two linear layers with ReLU/GELU between them, applied independently to each position.

3. Layer normalization and residual connections around each sub-layer.

No recurrence, no convolution — just attention and feed-forward layers. All positions are processed in parallel, making training much faster than RNNs.

Vision Transformers (ViT)

Dosovitskiy et al. (2020) applied Transformers directly to images: split image into 16×16 patches, flatten each patch into a vector, add positional embeddings, and process with a standard Transformer. With sufficient training data (JFT-300M or ImageNet-21K), ViT matches or exceeds CNNs.

Key insight: self-attention has global receptive field from layer 1, while CNNs build it gradually through layers of local convolutions. This allows ViT to capture long-range dependencies immediately but requires more data to learn the spatial structure that CNNs have built in.

Chapter 16

Transfer Learning

The most important practical technique — why you should almost never train from scratch.

Why Transfer Learning Exists

Training a deep network from scratch requires massive datasets (millions of images) and massive compute (hundreds of GPU-hours). Most real-world problems don't have millions of labeled images. Transfer learning solves this by reusing features learned on a large dataset (like ImageNet) and adapting them to your specific, smaller dataset.

🎯 The Core Idea

The early layers of a CNN learn generic, universal features: edges, textures, shapes, parts. These features are useful for almost any visual task. Only the final layers are task-specific (cat vs. dog, benign vs. malignant). So take a pretrained network, keep the early layers frozen, and only retrain the final layers on your data. This is transfer learning, and it works astonishingly well — even with only 100–1,000 training images.

How to Transfer Learn

Strategy 1: Feature Extraction

Freeze all conv layers. Remove the final FC layer. Replace it with a new FC layer matching your number of classes. Train only the new layer. Use when your dataset is very small and similar to ImageNet.

Strategy 2: Fine-Tuning

Start from pretrained weights. Replace the final layer. Train the entire network with a small learning rate (10–100× smaller than training from scratch). Optionally freeze early layers and only fine-tune later layers. Use when you have moderate data or your task differs from ImageNet.

How Similar Is Your Data?

Similar Data + Small Dataset

Example: classifying dog breeds (ImageNet contains dogs). → Feature extraction only. Train a linear classifier on top of frozen features.

Similar Data + Large Dataset

Example: object detection on everyday scenes. → Fine-tune the entire network with a small LR.

Different Data + Small Dataset

Example: classifying medical X-rays (very different from ImageNet). → Feature extraction from earlier layers (more generic). Risky — consider data augmentation.

Different Data + Large Dataset

Example: satellite imagery with millions of labeled patches. → Fine-tune entire network, possibly train from scratch if enough data.

Modern Transfer: Foundation Models

The latest paradigm goes beyond ImageNet pretraining. Models like CLIP (trained on 400M image-text pairs) learn visual concepts through natural language supervision. You can classify images with text descriptions, no training images needed ("zero-shot"). SAM (Segment Anything Model) segments any object in any image. These foundation models are pretrained on internet-scale data and can be adapted to virtually any visual task.

💡 The #1 Practical Takeaway

Unless you have millions of labeled images AND significant compute budget AND your domain is very different from natural images, always start with a pretrained model. In PyTorch: model = torchvision.models.resnet50(pretrained=True). Replace the last layer, fine-tune with a small learning rate. This simple recipe solves 90% of practical vision problems and is how virtually all real-world systems are built.

# Complete Transfer Learning Recipe in PyTorch import torch import torchvision.models as models # 1. Load pretrained model model = models.resnet50(weights='IMAGENET1K_V2') # 2. Freeze all layers for param in model.parameters(): param.requires_grad = False # 3.
